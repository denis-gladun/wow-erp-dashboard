# 📑 ПРОЕКТ: WoW Enterprise Resource Planning (ERP) Automation Dashboard

Модульная система сквозного операционного аудита, ETL-трансформации данных и мониторинга готовности исполнителей (бустеров) в реальном времени.

Система автоматизирует сбор данных из **Blizzard REST API**, обрабатывает стейт более **400 динамических профилей** персонажей и генерирует мгновенный UI-отчет для менеджмента (Рейд-лидеров), сокращая время проверки группы **с 10 минут до 5 секунд** (Оптимизация на 99.1%).

---

## 🏗️ Архитектура Системы (Паттерн MVC)

Проект спроектирован с чётким разделением слоев логики, что исключает перегрузку интерфейса и обеспечивает высокую отказоустойчивость:

* **Model (Слой Данных):** Табличные базы данных (`UI_Data`, `UI_Ilvl`, `UI_Gear`, `Raid_CD`). Сюда импортируются сырые JSON-ответы API.
* **Controller (Вычислительный слой):** Теневые листы-агрегаторы (`Booster_Data_Helper`, `техлист`). Выполняют нормализацию данных, парсинг регулярных выражений (`REGEX`), маппинг игровых классов и расчет КД.
* **View (Интерфейс):** Компактные панели управления (`Dashboard`, `RL_Dashboard`). Высокоинформативный фронтенд со «светофорной» системой индикации готовности всей рейдовой группы в один клик.

---

## ⚡ Ключевой Функционал и ETL-Пайплайн

Архитектура разделена на независимые воркеры, запускаемые по требованию (`On-Demand`). Это гарантирует высокую скорость обработки конкретных операционных задач и экономию квот API:

1. **Воркер Профилей:** Считывает конфигурацию и лимиты из листа управления, запрашивает базовый стейт персонажей и обновляет общую матрицу активов.
2. **Воркер Экипировки (Gear Audit):** Выполняет глубокий аудит каждого из 16 слотов снаряжения персонажа с проверкой наличия обязательных чантов, сокетов, специфических камней и лимитов на уникальные предметы (Embellishments).
3. **Воркер КД (Cooldown Monitor):** Экстрагирует данные из `Raid_CD`, опрашивает Blizzard API на предмет заблокированных рейдовых сохранений и выгружает обновленные статусы в настроенные целевые диапазоны для оптимизации логистики запусков.
4. **Воркеры Логистики и Распределения Комплектов (Voidspire / Dreamrift Matrix):** Высокоуровневые модули автоматического расчета и генерации оптимальных рейдовых составов на основе iLvl, КД и минимизации трения кросс-брони (`cross-armor friction`) по каждому из 16 слотов экипировки.

## 🌐 Облачная Интеграция и Асинхронный Движок (Local API + ngrok)

Для перевода проекта из локального консольного скрипта в распределенную облачную ERP-систему реализована сетевая инфраструктура шлюзов:

* **Серверное туннелирование (ngrok):** Локальный вычислительный Python-сервер проброшен в веб через безопасный туннель `ngrok`, разворачивая доступный для внешних вебхуков API-эндпоинт.
* **Интеграция с Google Apps Script (GAS):** В облачный интерфейс таблиц добавлены интерактивные макрос-кнопки. При клике рейд-лидером, скрипт GAS отправляет асинхронный HTTP-запрос на туннель `ngrok`.
* **Автономность менеджмента:** Координаторы полностью изолированы от консоли — тяжелые алгоритмы распределения и обновления данных запускаются одной кнопкой напрямую из Google Sheets.
* **Динамический No-Code Конфигуратор:** Логические условия подбора (лимиты ролей, пулы мейнов/сабов, пороговые iLvl, флаги проверки КД) полностью вынесены из хардкода скриптов в интерфейсные шапки листов вывода (`Voidspire_Output` / `Dreamrift_Output`). Скрипт считывает параметры «на лету» перед генерацией.

## 🛠️ Технологический Стек

* **Backend / ETL:** Python 3.x, Requests (синхронизация с Blizzard REST API, Warcraft Logs API), `oauth2client`, асинхронные обработчики.
* **Инфраструктура и Сеть:** `ngrok` (Secure Tunneling), Google Apps Script (JavaScript) для обработки UI-событий.
* **Database / Warehouse:** Google Sheets API (`gspread`) как гибкое, реляционно-связанное облачное хранилище данных.
* **Data Processing:** Sheets Complex Logic (ArrayFormula, REGEX-парсинг регулярных выражений, ДВССЫЛ, условное форматирование).

## 🚀 Спецификация Запуска (On-Demand Execution)

Скрипты синхронизации могут запускаться как через облачные кнопки интерфейса (посредством связки GAS + ngrok), так и вручную из консоли администратора:

```bash
# Обновление базовых метрик и iLvl профилей
python sync_profiles.py

# Запуск точечного аудита чантов, сокетов и камней
python sync_gear.py

# Проверка и актуализация рейдовых сохранений (Raid CD)
python sync_cooldowns.py

# Generation логистической матрицы и дашборда для Voidspire Mythic
python ВОИДСПАЙР_ЛОГИСТИКА.py

# Generation логистической матрицы и дашборда для Dreamrift Mythic
python ДРИМРИФТ_ЛОГИСТИКА.py
```
